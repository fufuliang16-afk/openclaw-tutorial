OpenClaw使用教程

从基础到高级，全面掌握OpenClaw的使用方法。

快速开始

第一步：启动OpenClaw

安装完成后，你可以通过以下方式启动OpenClaw：

方式1：命令行模式
# 直接运行命令
openclaw "帮我查看当前目录的文件"

# 或者进入交互模式
openclaw
方式2：作为服务运行
# 启动网关服务
openclaw gateway start

# 检查服务状态
openclaw gateway status

# 查看日志
openclaw logs --follow
方式3：通过API调用
# OpenClaw提供REST API
curl -X POST http://localhost:18789/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "你好"}'

基本命令

命令 描述 示例
openclaw --help 显示帮助信息 openclaw --help
openclaw --version 显示版本信息 openclaw --version
openclaw status 检查系统状态 openclaw status
openclaw doctor 诊断系统问题 openclaw doctor --fix
openclaw configure 运行配置向导 openclaw configure
openclaw channels list 列出通信渠道 openclaw channels list
提示：大多数命令都支持 --help 参数查看详细用法。

文件操作

OpenClaw可以帮你处理各种文件操作任务：

读取文件

你可以让OpenClaw读取和分析文件：

# 直接请求读取文件
openclaw "请读取README.md文件并总结内容"

# 或者更具体的指令
openclaw "分析package.json文件中的依赖"
编辑文件

OpenClaw可以智能编辑文件：

# 添加内容到文件
openclaw "在README.md末尾添加安装说明"

# 修改文件内容
openclaw "更新config.json中的API密钥"

# 创建新文件
openclaw "创建一个Python脚本用于数据清洗"
文件操作示例
示例1：整理项目文档
# 让OpenClaw整理项目文档
openclaw "请扫描当前目录的所有Markdown文件，创建一个索引文档"
示例2：批量重命名
# 批量重命名文件
openclaw "将所有.jpg文件重命名为image_001.jpg格式"
示例3：数据提取
# 从日志文件中提取信息
openclaw "从app.log中提取所有错误信息并保存到errors.txt"

网络搜索与信息获取

OpenClaw可以帮你搜索网络信息：

基本搜索
# 搜索最新信息
openclaw "搜索OpenAI最新发布的模型"

# 获取技术文档
openclaw "查找Node.js v20的新特性"

# 研究某个主题
openclaw "研究机器学习在医疗领域的应用"
高级信息处理
# 比较分析
openclaw "比较React和Vue.js的优缺点"

# 数据收集
openclaw "收集2024年AI领域的重要事件"

# 趋势分析
openclaw "分析当前前端开发的技术趋势"
注意：网络搜索功能需要配置相应的API密钥，请参考安装指南中的配置步骤。

记忆管理

OpenClaw使用文件系统来管理记忆，确保会话间的连续性：

记忆文件结构
~/.openclaw/workspace/
├── MEMORY.md          # 长期记忆
├── memory/            # 每日记忆
│   ├── 2024-01-01.md
│   ├── 2024-01-02.md
│   └── ...
├── AGENTS.md          # 代理配置
├── SOUL.md            # 个性定义
└── USER.md            # 用户信息
记忆操作
# 查看记忆
openclaw "我们昨天讨论了什么？"

# 更新记忆
openclaw "记住我喜欢用VS Code编辑器"

# 整理记忆
openclaw "整理最近一周的工作记录"

OpenClaw会自动：

  • 记录重要对话
  • 保存用户偏好
  • 维护项目上下文
  • 创建每日总结

高级功能

OpenClaw可以创建子代理来处理复杂任务：

# 创建专门处理数据的子代理
openclaw "创建一个子代理来分析销售数据"

# 并行处理多个任务
openclaw "同时搜索Python教程和JavaScript教程"
子代理可以独立运行，适合处理耗时任务或需要专门知识的任务。

创建自动化工作流：

# 每日自动化任务
openclaw "设置每日9点检查邮件和日历"

# 数据处理流水线
openclaw "创建数据处理流水线：下载→清洗→分析→报告"

# 监控任务
openclaw "监控服务器日志，发现错误时通知我"

扩展OpenClaw功能：

# 安装插件
openclaw plugins install @openclaw/feishu

# 列出已安装插件
openclaw plugins list

# 配置插件
openclaw "配置飞书插件连接到我的账号"

常用插件：

  • 飞书插件 - 企业通信集成
  • Git插件 - 版本控制
  • 日历插件 - 日程管理
  • 邮件插件 - 邮件处理
